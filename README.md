# YouMusicFlow

A compact, frameless, tray-based mini-player that wraps [YouTube Music](https://music.youtube.com) in a lightweight Electron window. Lives in your system tray — click to show, click away to hide.

![Platform](https://img.shields.io/badge/platform-macOS%20%7C%20Windows%20%7C%20Linux-blue)
![License](https://img.shields.io/badge/license-MIT-green)

## Features

- **System tray player** — sits quietly in your tray, pops open on click
- **Frameless & compact** — 375×667px window, no browser chrome
- **Media key support** — play/pause, next, previous via keyboard media keys
- **Ad & tracker blocking** — blocks doubleclick, Google Analytics, and other ad domains
- **Auto-updating** — checks for and installs updates automatically
- **Highly configurable** — always on top, start on login, window position, and more via Settings
- **Dark/light tray icon** — adapts to your macOS appearance on the fly
- **Cross-platform** — macOS, Windows (NSIS + Microsoft Store), Linux

## Install

Download the latest release for your platform from the [Releases](https://github.com/everancii/youmus/releases) page.

### macOS

1. Download the `.dmg` file
2. Open it and drag the app to your Applications folder
3. Launch — the app will appear in your menu bar

### Windows

- **Installer:** Download and run the NSIS setup executable
- **Microsoft Store:** Available on the Microsoft Store (search "YouMusicFlow")

### Linux

Download the AppImage or package for your distribution from Releases.

## Usage

| Action | What happens |
|---|---|
| **Click tray icon** | Opens the player window |
| **Click outside window** | Hides the player |
| **Right-click tray icon** | Opens context menu (Settings, Quit) |
| **Media keys** | Play/Pause, Next, Previous |
| **Ctrl/Cmd + Q** | Quit the app |

### Settings

Right-click the tray icon and choose **Settings** to configure:

- **Always on Top** — keep the player above other windows
- **Start on Login** — launch automatically when you log in
- **Window Position** — choose a corner or let the app auto-detect based on tray position

## Build from Source

### Prerequisites

- [Node.js](https://nodejs.org) (v18+)
- npm

### Setup

```bash
git clone https://github.com/everancii/youmus.git
cd youmus
npm install
```

### Commands

| Command | Description |
|---|---|
| `npm start` | Build and launch the app |
| `npm run build` | Compile TypeScript and copy settings assets |
| `npm run watch` | Watch for changes and recompile |
| `npm test` | Run the test suite |
| `npm run test:coverage` | Run tests with coverage report |
| `npm run format` | Format code with Prettier |
| `npm run dist` | Package distributable (`release/`) |
| `npm run pack` | Package into a directory (no installer) |

### Development

1. Run `npm run watch` in one terminal to recompile on change
2. Run `npm start` in another to launch the app

> **Note:** The `app/` directory is auto-generated by the build. Never edit files there directly.

## Architecture

```
main.ts                          Electron main process
├── client/index.ts              Preload script (context isolation)
│   └── controls.ts              DOM queries on YouTube Music
├── tools/
│   ├── auto-update.ts           Self-updater (electron-updater)
│   ├── settings.ts              Persistent config (electron-store)
│   ├── platformResolver.ts      OS detection
│   ├── getTrayPosition.ts       Tray position detection
│   └── offsetCalclator.ts       Per-OS pixel offsets
├── ui/
│   ├── menuTemplate.ts          Application menu bar
│   └── contextTemplate.ts       Tray context menu
├── settings/                    Settings window (HTML/CSS/JS)
│   ├── index.html
│   ├── renderer.js
│   └── style.css
└── types/ipc.ts                 IPC event type definitions
```

**Media key flow:**

```
OS media key → globalShortcut → IPC → preload → controls.ts → YouTube Music DOM
```

## Tech Stack

- [electron-updater](https://github.com/electron-userland/electron-builder) — auto-updates

- [Prettier](https://prettier.io/) — code formatting
